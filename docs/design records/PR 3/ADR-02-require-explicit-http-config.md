# ADR-02: Явное требование client.http без автодефолтов

## Контекст
- Ранее `ClientConfig` подставлял новый `HttpClientConfig`, если блок `client.http` отсутствовал в YAML.
- Это скрывало ошибки конфигурации и усложняло прогнозируемость таймаутов/ретраев.

## Решение
- Убрано автозаполнение `HttpClientConfig` при `http == null`.
- Валидатор требует наличия `client.http`; при отсутствии бросается `ConfigValidationException`.
- Тесты (`ConfigLoaderTest`, `ConfigBranchTest`) ожидают ошибку при пропущенном http-блоке.

## Последствия
- Конфиги должны явно задавать HTTP-настройки; отсутствующий блок считается ошибкой.
- Риск скрытых изменений дефолтов устранён; поведение конфигурации прозрачнее.
- Требуется обновить пользовательские YAML, чтобы включать `client.http`.
